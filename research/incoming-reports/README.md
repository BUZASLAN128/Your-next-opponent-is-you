# Incoming Research Reports

This directory preserves user-supplied research outputs before their claims are
promoted into the project's evidence base.

Files here are **inputs, not authority**. A polished explanation, citation
marker, mathematical expression, architecture diagram, or named framework does
not by itself establish a fact or decision.

For each report:

- preserve the supplied text and its content hash;
- label the source and intake date;
- keep recommendations and infrastructure choices unconfirmed;
- reconstruct missing citations before relying on external claims;
- record contrary evidence and transfer limits;
- move distilled findings into topic documents only with source-linked status;
- add explicit user decisions to the decision log rather than inferring them
  from the report.

If a supplied report contains raw conversation, direct identity-bearing spans,
third-party personal data, credentials, or another Git-prohibited class, the
privacy boundary overrides verbatim preservation. Keep the original outside
Git and preserve only its content hash, provenance, redacted claim map, and the
reason verbatim intake was blocked.

The current comparative assessment is in
[the first report review](../report-review-2026-07-15.md).

The first report from the next deep-research batch is reviewed in
[the Round 1 synthesis](../deep-research-round-01-synthesis-2026-07-15.md) and
integrated into the [living convergence map](../convergence-map.md).

The second report from that batch is reviewed in
[the Round 2 synthesis](../deep-research-round-02-synthesis-2026-07-15.md). It
is retained as a stronger ontology-and-evaluation input, not a proven schema or
promotion policy.
